Healthcare Chatbot Architecture

Rule-Based + LLM Hybrid Response System

1. Message Flow (Overall)

flowchart TD
    User([User Message]) --> FE[Frontend chat.html]
    FE -->|POST /api/chat/stream| Router[chat.py Router]
    Router --> CS[ChatService.stream_chat]
    CS --> Save1[(Save user msg to Supabase)]
    Save1 --> RE{RuleEngine.try_respond}

    RE -->|Match| RuleResp["Return (response, function_name)"]

    Save2 --> Badge{Frontend Badge Display}

RULE Rule-Based Functions (4)

  • BMI 계산기 — 키/몸무게로 BMI 계산 + 판정
  • "bmi", "체질량" + cm/kg 숫자
  • 칼로리 조회 — 25개 음식 DB에서 즉시 조회
  • "칼로리", "열량", "kcal" + 음식 이름
  • 건강 FAQ — 6개 주제 사전 답변
  • "간헐적단식", "물 섭취량", "스트레칭", "수면", "단백질", "기초대사량"
  • 인사말 — 짧은 인사 자동 응답
  • "안녕", "하이", "hello", "hi" (15자 이내)

LLM LLM Fallback

  • 조건 — Rule Engine이 None 반환 시
  • 모델 — OpenRouter (GPT-4o-mini 기본)
  • 컨텍스트 — System Prompt + Agent Memory + Chat History
  • 추가 처리 — Memory Extraction + Context Summary
  • 비용 — API 호출 비용 발생
  • 지연 — 1~5초 (모델/길이에 따라)

DATA Data Flow

  • Supabase — chat_messages, chat_sessions 저장
  • agent_memory — LLM 응답 시만 memory 추출
  • context_summary — 12+ 메시지 시 요약 생성
  • SSE Stream — source + function 필드 포함
  • Frontend Badge — 초록(rule) / 보라(LLM) 배지
