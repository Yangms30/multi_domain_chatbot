|------|------|

| 시작일 | 2026-03-27 |
| 목표 | "인터스텔라 분석해줘" → DB 영화 데이터 + LLM 전문 분석 응답 |

### Value Delivered

| 관점 | 내용 |
|------|------|
| **Problem** | 현재 영화 챗봇은 정보 조회(제목/감독/평점)만 가능. 영화를 깊이 있게 분석하는 기능이 없어서 "영화 덕후"를 위한 차별화 포인트 부재 |
| **Solution** | "분석해줘" 키워드 감지 시, DB에서 영화 기본 정보를 가져와 LLM context로 주입하고, 전문 분석 시스템 프롬프트로 촬영기법/스토리/테마 분석 생성 |
| **Function UX Effect** | 영화 전문가 수준의 분석 (촬영기법, 서사 구조, 상징, 테마 해석), DB 데이터로 hallucination 최소화 |
| **Core Value** | 단순 추천 봇 → 영화 전문 분석 봇으로 진화, "영화 버프" 에이전트의 핵심 차별점 |

## 1. 배경

### 현재 LLM 호출 구조
```
User Message
  → RuleEngine (rule + BM25) → 응답 성공 시 LLM 비용 0원
  → (실패) → LLM fallback (일반 system prompt로 응답)
```

### 문제점
- LLM fallback은 영화 도메인 system prompt만 가지고 응답 → 일반적인 답변
- DB에 있는 영화 상세 데이터(감독, 캐스트, 장르, 줄거리)를 활용하지 않음
- "분석해줘" 같은 전문 분석 요청에 대한 전용 처리 없음

### Film Analysis의 차별점
기존 LLM fallback과 다른 점:
1. **DB 데이터를 context로 주입** — hallucination 방지
2. **분석 전용 system prompt** — 촬영기법, 서사 구조 등 전문 영역 가이드
3. **구조화된 응답** — 분석 카테고리별 섹션 (일반 대화가 아닌 분석 레포트)

## 2. 구현 범위

### Task 1: 분석 요청 감지 + DB 데이터 조회
- **수정 파일**: `backend/services/knowledge_query.py`
- `_try_film_analysis()` 메서드 추가
- 키워드: "분석", "해석", "촬영기법", "스토리 구조", "테마", "상징", "연출"
- 영화 제목 추출 → DB에서 기본 정보 조회 → context 생성

### Task 2: 분석 전용 LLM 호출
- **신규 파일**: `backend/services/film_analysis_service.py`
- DB 데이터를 기반으로 분석 전용 system prompt + user prompt 구성
- OpenRouter를 통한 LLM 호출 (스트리밍)
- 분석 카테고리:
  1. **촬영/연출** — 촬영 기법, 시각적 스타일, 특수효과
  2. **스토리/서사** — 3막 구조, 플롯 포인트, 서브플롯
  3. **테마/상징** — 핵심 메시지, 상징적 요소, 사회적 의미
  4. **연기/캐릭터** — 캐릭터 아크, 주요 연기 평가
  5. **총평** — 영화사적 의미, 추천 대상

### Task 3: 응답 통합
- `chat_service.py` 수정: film analysis 요청 감지 시 전용 파이프라인으로 분기
- 스트리밍 응답 지원
- source badge: "영화 전문 분석 (AI)"

## 3. 기술 결정

### LLM 비용 최적화
| 전략 | 설명 |
|------|------|
| **DB context 주입** | 영화 기본 정보(감독, 장르, 줄거리 등)를 system prompt에 포함 → LLM이 정확한 데이터 기반으로 분석 |
| **분석 전용 prompt** | 일반 대화가 아닌 구조화된 분석을 요청 → 토큰 낭비 최소화 |
| **max_tokens 제한** | 분석 응답 max 1500 토큰 (약 500-700 한글자) |
| **Rule-based 우선** | "기생충 알려줘"는 기존 DB 응답, "기생충 분석해줘"만 LLM 호출 |

### 비용 예상
- 모델: `openai/gpt-4o-mini` (현재 설정)
- 입력: ~800 토큰 (system prompt 500 + DB context 200 + user 100)
- 출력: ~1000 토큰
- 비용: 약 $0.0003/요청 (gpt-4o-mini 기준)

## 4. 분석 System Prompt (핵심)

```
당신은 영화 전문 분석가입니다. 아래 영화 정보를 바탕으로 전문적인 분석을 제공해주세요.

[영화 정보]
제목: {title}
감독: {director}
장르: {genres}
개봉: {release_date}
줄거리: {overview}
출연: {cast}

다음 항목에 대해 분석해주세요:

## 촬영/연출 분석
감독의 연출 스타일, 촬영 기법, 시각적 특징을 분석해주세요.

## 스토리/서사 구조
서사 구조, 주요 플롯 포인트, 전개 방식을 분석해주세요.

## 테마/상징
핵심 테마, 상징적 요소, 사회적/철학적 메시지를 해석해주세요.

## 총평
영화사적 의미와 이 영화를 추천할 대상을 정리해주세요.

규칙:
- 한국어로 답변
- 각 섹션 2-3문장
- 스포일러 주의 표시
- 주관적 의견과 객관적 사실을 구분
```

## 5. 구현 순서

```
[1] knowledge_query.py — _try_film_analysis() 키워드 감지 + 영화 제목 추출

[2] film_analysis_service.py — 분석 prompt 구성 + LLM 호출
 ↓
[3] chat_service.py — film analysis 파이프라인 통합 (스트리밍)
```

## 6. 성공 기준

- [ ] "기생충 분석해줘" → 4개 카테고리 구조화된 분석 응답
- [ ] DB에 있는 영화: 정확한 감독/장르/줄거리 기반 분석 (hallucination 없음)
- [ ] DB에 없는 영화: LLM 자체 지식으로 분석 (데이터 부재 안내 포함)
- [ ] "인터스텔라 촬영기법" → 촬영/연출 섹션만 집중 분석
- [ ] 스트리밍 응답 동작
- [ ] source badge "영화 전문 분석 (AI)" 표시
- [ ] 기존 "기생충 알려줘" (정보 조회)와 충돌 없음
